# የአይሲቲ ድጋፍ የስራ ሂደት / ICT Support Workflow

## የስራ ሂደት / Workflow Process

### 1️⃣ ተጠቃሚ ጥያቄ ያስገባል / User Submits Request

**ተጠቃሚ:**
- ወደ አይሲቲ ድጋፍ ገጽ ይሄዳል
- "አዲስ ድጋፍ ጥያቄ" ይጫናል
- ፎርሙን ይሞላል:
  - ርዕስ (Subject)
  - ዝርዝር መግለጫ (Description)
  - ምድብ (Category)
  - ቅድሚያ (Priority)
- "ጥያቄ ላክ" ይጫናል

**ሁኔታ:** 🟡 PENDING (በመጠባበቅ ላይ)

---

### 2️⃣ አይቲ አስተዳዳሪ ጥያቄውን ያያል / IT Admin Views Request

**አይቲ አስተዳዳሪ:**
- ወደ አይሲቲ ድጋፍ ገጽ ይሄዳል
- "ሁሉም የድጋፍ ጥያቄዎች" ክፍል ያያል
- ጥያቄዎች በቅድሚያ ደረጃ ተደርደዋል:
  - 🔴 URGENT (አስቸኳይ)
  - 🟠 HIGH (ከፍተኛ)
  - 🟡 MEDIUM (መካከለኛ)
  - 🟢 LOW (ዝቅተኛ)

---

### 3️⃣ አይሲቲ ሙያተኛ ስራ ይጀምራል / ICT Professional Starts Work

**አይቲ አስተዳዳሪ:**
- "🚀 ስራ ጀምር" ቁልፍን ይጫናል
- ማረጋገጫ መልእክት ይታያል
- "OK" ይጫናል

**የሚከሰቱ ነገሮች:**
- ሁኔታ ወደ "በሂደት ላይ" ይቀየራል
- አይሲቲ ሙያተኛ ይመደባል
- ተጠቃሚው ማሳወቂያ ይቀበላል: "አይሲቲ ሙያተኛ ወደ ቦታው በመሄድ ላይ ነው..."

**ሁኔታ:** 🔵 IN PROGRESS (በሂደት ላይ)

---

### 4️⃣ አይሲቲ ሙያተኛ ወደ ቦታው ይሄዳል / ICT Professional Goes to Location

**አይሲቲ ሙያተኛ:**
1. ወደ ተጠቃሚው ቦታ ይሄዳል
2. ችግሩን ይመረምራል
3. ችግሩን ይፈታል
4. መፍትሄውን ይፈትሻል

**በዚህ ጊዜ:**
- ሁኔታ "በሂደት ላይ" ሆኖ ይቀራል
- ተጠቃሚው "አይሲቲ ሙያተኛ በስራ ላይ ነው..." መልእክት ያያል

---

### 5️⃣ አይሲቲ ሙያተኛ መልስ ይሰጣል / ICT Professional Provides Response

**አይቲ አስተዳዳሪ:**
- ወደ አይሲቲ ድጋፍ ገጽ ይመለሳል
- "💬 መልስ ስጥ" ቁልፍን ይጫናል
- የመልስ ፎርሙን ይሞላል:
  - **ሁኔታ:** ተፈትቷል (Resolved) ወይም ተዘግቷል (Closed)
  - **መፍትሄ:** ዝርዝር መግለጫ ያስገባል
- "መልስ ላክ" ይጫናል

**የመፍትሄ ምሳሌ:**
```
ችግሩ የነበረው የኮምፒውተሩ RAM ስለተበላሸ ነው። 
አዲስ RAM ተገዝቶ ተተክሏል። 
ኮምፒውተሩ አሁን በትክክል እየሰራ ነው።
የተወሰደ እርምጃ:
- የድሮውን RAM አውጥቻለሁ
- አዲስ 8GB RAM ጫንኩ
- ስርዓቱን ፈትሼ ሙሉ በሙሉ እየሰራ ነው
```

**ሁኔታ:** 🟢 RESOLVED (ተፈትቷል)

---

### 6️⃣ ተጠቃሚ መልሱን ያያል / User Views Response

**ተጠቃሚ:**
- ማሳወቂያ ይቀበላል
- ወደ አይሲቲ ድጋፍ ገጽ ይሄዳል
- የመፍትሄውን ዝርዝር ያያል
- ችግሩ መፈታቱን ያረጋግጣል

---

## የሁኔታ ለውጦች / Status Changes

### 🟡 PENDING → 🔵 IN PROGRESS
**መቼ:** አይሲቲ ሙያተኛ "ስራ ጀምር" ሲጫን  
**ማሳወቂያ:** "አይሲቲ ሙያተኛ ወደ ቦታው በመሄድ ላይ ነው..."

### 🔵 IN PROGRESS → 🟢 RESOLVED
**መቼ:** አይሲቲ ሙያተኛ ችግሩን ከፈታ በኋላ መልስ ሲሰጥ  
**ማሳወቂያ:** "የአይሲቲ ድጋፍ ጥያቄዎ ተፈትቷል። መፍትሄ: ..."

### 🔵 IN PROGRESS → ⚫ CLOSED
**መቼ:** ችግሩ መፈታት ካልቻለ ወይም ተጠቃሚው ጥያቄውን ሰርዞ  
**ማሳወቂያ:** "የአይሲቲ ድጋፍ ጥያቄዎ ተዘግቷል።"

---

## የቁልፍ ተግባራት / Button Functions

### 🚀 ስራ ጀምር (Start Work)
- **የሚታይበት:** በ PENDING ጥያቄዎች ላይ
- **ተግባር:** ሁኔታን ወደ IN PROGRESS ይቀይራል
- **ማሳወቂያ:** ተጠቃሚው ይቀበላል

### 💬 መልስ ስጥ (Respond)
- **የሚታይበት:** በ PENDING እና IN PROGRESS ጥያቄዎች ላይ
- **ተግባር:** መፍትሄ ይሰጣል እና ሁኔታ ይቀይራል
- **ማሳወቂያ:** ተጠቃሚው ይቀበላል

---

## ምሳሌ ሁኔታ / Example Scenario

### ሁኔታ: ኮምፒውተር አይነሳም

**1. ተጠቃሚ ጥያቄ ያስገባል:**
- ርዕስ: "ኮምፒውተሩ አይነሳም"
- መግለጫ: "የቢሮዬ ኮምፒውተር ዛሬ ጠዋት ከተነሳ በኋላ እንደገና አይነሳም"
- ምድብ: Hardware
- ቅድሚያ: High
- ⏰ 9:00 AM

**2. አይቲ አስተዳዳሪ ጥያቄውን ያያል:**
- ጥያቄው በ "ሁሉም የድጋፍ ጥያቄዎች" ክፍል ይታያል
- ቅድሚያ HIGH ስለሆነ በላይ ይታያል
- ⏰ 9:05 AM

**3. አይሲቲ ሙያተኛ ስራ ይጀምራል:**
- "🚀 ስራ ጀምር" ይጫናል
- ሁኔታ: IN PROGRESS
- ተጠቃሚ ማሳወቂያ ይቀበላል
- ⏰ 9:10 AM

**4. አይሲቲ ሙያተኛ ወደ ቦታው ይሄዳል:**
- ወደ ተጠቃሚው ቢሮ ይሄዳል
- ኮምፒውተሩን ይመረምራል
- ችግሩ የኤሌክትሪክ ኬብል መበላሸት መሆኑን ያገኛል
- አዲስ ኬብል ይዞ ይመጣል
- ኬብሉን ይቀይራል
- ኮምፒውተሩ መስራቱን ያረጋግጣል
- ⏰ 9:30 AM - 10:00 AM

**5. አይሲቲ ሙያተኛ መልስ ይሰጣል:**
- ወደ ቢሮው ይመለሳል
- "💬 መልስ ስጥ" ይጫናል
- ሁኔታ: Resolved
- መፍትሄ: "ችግሩ የነበረው የኤሌክትሪክ ኬብል ስለተበላሸ ነው። አዲስ ኬብል ተተክሏል። ኮምፒውተሩ አሁን በትክክል እየሰራ ነው።"
- ⏰ 10:15 AM

**6. ተጠቃሚ መልሱን ያያል:**
- ማሳወቂያ ይቀበላል
- መፍትሄውን ያነባል
- ኮምፒውተሩ መስራቱን ያረጋግጣል
- ⏰ 10:20 AM

**ጠቅላላ ጊዜ:** 1 ሰዓት 20 ደቂቃ ✅

## ምክሮች ለአይሲቲ ሙያተኞች / Tips for ICT Professionals

### ስራ ሲጀምሩ:
1. ✅ ጥያቄውን በጥንቃቄ ያንብቡ
2. ✅ የሚያስፈልጉ መሳሪያዎችን ይዘው ይሂዱ
3. ✅ "ስራ ጀምር" ቁልፍን ይጫኑ (ተጠቃሚው እንዲያውቅ)

### ችግሩን ሲፈቱ:
1. ✅ ችግሩን በደንብ ይመርምሩ
2. ✅ መፍትሄውን ይፈትሹ
3. ✅ ተጠቃሚውን ያስተምሩ (ችግሩ እንዳይደገም)

### መልስ ሲሰጡ:
1. ✅ ግልጽ መግለጫ ይስጡ
2. ✅ የተወሰዱ እርምጃዎችን ይዘርዝሩ
3. ✅ ተጨማሪ ምክሮችን ይስጡ

## የተለመዱ ችግሮች እና መፍትሄዎች / Common Issues and Solutions

### ችግር 1: ኮምፒውተር ዘግይቷል
**መፍትሄ:**
- RAM ይፈትሹ
- Disk space ይፈትሹ
- Virus scan ያድርጉ
- Startup programs ይቀንሱ

### ችግር 2: ኢንተርኔት አይገናኝም
**መፍትሄ:**
- ኬብል ግንኙነት ይፈትሹ
- Router ዳግም ያስጀምሩ
- IP configuration ይፈትሹ
- DNS settings ይቀይሩ

### ችግር 3: ፕሪንተር አይሰራም
**መፍትሄ:**
- ኬብል ግንኙነት ይፈትሹ
- Driver ይፈትሹ
- Print queue ያጽዱ
- ፕሪንተር ዳግም ያስጀምሩ
